# Governance Toolkit

| Framework | Use |
|---|---|
| SOC2 | Security and control effectiveness evidence |
| ISO/IEC 27001 | ISMS control mapping and governance |
| NIST AI RMF | AI risk lifecycle governance |
| OWASP LLMSVS | LLM control verification requirements |

## Open-Source Tooling

| Tool | Use |
|---|---|
| [Open Policy Agent](https://github.com/open-policy-agent/opa) | Policy-as-code for governance decisions and release checks |
| [Open Controls](https://github.com/opencontrol) | Structured control library and automation-friendly evidence mapping |
| [Compliance as Code](https://github.com/opencontrol/compliance-as-code) | Policy packs and control testing workflows |
| [Grading Framework](https://github.com/govready) | Risk and control maturity assessments with OSS workflows |
| [Inspec](https://github.com/inspec/inspec) | Compliance evidence auditing and verification as tests |
| [OpenSCAP](https://github.com/OpenSCAP/openscap) | Security benchmarks and policy baseline checks |
| [CycloneDX CLI](https://github.com/CycloneDX/cyclonedx-cli) | SBOM generation and evidence artifact validation |
| [In-toto](https://github.com/in-toto/in-toto) | Supply-chain provenance and attestations for release evidence |
| [CISO Assistant](https://github.com/ciso-assistant/ciso-assistant) | GRC workflows, risk registers, exception tracking |

## Execution Pattern
1. Map controls to required frameworks.
2. Define evidence artifacts per release and incident.
3. Track exceptions with expiry and compensating controls.
